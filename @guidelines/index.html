本ガイドラインについて

本ガイドラインは、Webサイト制作を行う際の記述ルール等を定めたものです。
クオリティの向上・均一化、制作の効率化を目的とし、常に見直しを行い、必要に応じ検討・更新を行います。
また、本ガイドラインが適用されるのは、新規サイトのみです。
以前に制作したサイトや、既にガイドラインがある場合はそのルールに準じます。

サイトの仕様

本ガイドラインでは、Webサイトの基本仕様を以下の指定で制作する事と定めます。
ただし、案件ごとに違うこともありますので、サイト作成前に必ず確認してください。

DTD HTML5
CSS CSS2.1〜
文字コード UTF-8
改行コード CR+LF
対象ブラウザ※要確認 PC:[ Safari/Firefox/Chrome/MS Edgeの各最新ブラウザ]
iphone、android、ipad

※CSSのバージョンについて（https://wemo.tech/807#index_id4）

命名規則

  • 半角英数字のみを使用する。
  • 初めの文字はアルファベットとする。
  • 全角・半角スペースは使用しない。
  • 単語が複数連続する場合は、単語間に「-（ハイフン）」を使用する。（画像の場合は「_（アンダーバー）」）

ディレクトリ名

  • 動的の場合は複数形（s）が付くようにする。
  • カテゴリーを英語化したものを使用する。
    例）概要 outline/、イベント event/、九州地図 map-kyushu/
  • 単語が複数連続する場合は、単語間に「-（ハイフン）」を使用する。
  • ページタイトルが複数の単語でファイル名が長くなる場合は、単語を1〜2語に絞って命名する。
    例）2015年会社沿革 history2015/
  • 同様のディレクトリが複数存在する場合は連番を使用可能とする。
    例）店舗詳細 shop001/、shop002/ ...

HTMLマークアップのルール

HTMLの本質は文書の構造化であり、マークアップはそのための手段です。
正しいマークアップを行なうためにはHTMLタグ一つひとつの意味を理解し適切に判断して使用しなければいけません。
そのためのガイドラインを以下に示します。

書式ルール

  • 文書構造・見た目・振る舞いは、分離すること。
  • HTMLファイル内では基本的にHTMLのみのマークアップ行い、視覚表現（装飾やアニメーション）は外部ファイル化したCSS、JavaScriptで行うこと。
  • アクセシビリティの観点からhtml内にstyleタグは使用しない。

プロトコル

  • 可能であれば、埋め込みリソースにはHTTPSプロトコルを使用してください。
  • それぞれのファイルがHTTPS上で利用可能でない限り、イメージやその他のメディアファイル、スタイルシート、およびスクリプトには常にHTTPSプロトコル（https :)を使用します。
  • 古いページを流用したときに、外部から読み込まれているファイルがHTTPSになっているか必ず確認してください。

	<!-- OK -->
	<script src="https://ajax.googleapis.com/ajax/libs/jquery/3.1.0/jquery.min.js"></script>

	<!-- NG -->
	<script src="//ajax.googleapis.com/ajax/libs/jquery/3.1.0/jquery.min.js"></script>

	<!-- NG -->
	<script src="http://ajax.googleapis.com/ajax/libs/jquery/3.1.0/jquery.min.js"></script>

	/* OK */
	@import 'https://fonts.googleapis.com/css?family=Open+Sans';

	/* NG */
	@import '//fonts.googleapis.com/css?family=Open+Sans';

	/* NG */
	@import 'http://fonts.googleapis.com/css?family=Open+Sans';

画像のalt

  • 画像には、意味のある代替テキストをalt属性として、動画・オーディオコンテンツにはキャプションを記述する。
  • 装飾的な用途の場合など意味を持たない画像については、代替テキストは記述せずにalt=""とする。
  • 以下のように、alt名の文頭に付ける名前を統一するとよい。
    • 写真の画像の場合は、「alt="写真：○○"」
    • イラストの画像の場合は、「alt="イラスト：○○"」
    • 図の画像の場合は、「alt="図：○○"」
    • ※ 「：」の文字は全角にする。（半角の「:」はmichekerでエラーが出る）

パスの指定ルール

  • サイト内リンクは指定が無い場合、原則として相対パスとする。

	<!-- 相対パス -->
	<a href="../company/">リンク</a>

	<!-- ルートパス -->
	<img src="/img/ttl-main.gif">

	<!-- 絶対パス -->
	<a href="https://www.kiml.co.jp/outline/" target="_blank">リンク</a>
  • PDFファイルへのリンク時は、target属性に「_blank」を指定する。
  • 外部サイトへのリンク時は、target属性に「_blank」を指定する。

	<!--  PDFファイルへのリンク -->
	<a href="/pdf/example.pdf" target="_blank">PDFドキュメント</a>

	<!-- 外部サイトへのリンク -->
	<a href="https://www.kiml.co.jp/outline/" target="_blank">リンク</a>
  • リンク先が不明のところは開発にわかりやすいよう「#」を入れる。

	<!-- indexファイルへのリンク -->
	<a href="#">サイトマップ</a>

インデント

  • タブでインデントする。
  • 半角スペースを使ったり、タブとスペースを混在させるのはNG

	<ul>
		<li>Fantastic</li>
		<li>Great</li>
	</ul>
  • ブロック要素・リスト要素・テーブル要素は改行してから記述し、それらの子要素にはインデントを入れる。
  • 横並びリストなど改行による空白が問題になる場合は、li要素をすべて一行で書いてもOK

	<blockquote>
		<p><em>Space</em>, the final frontier.</p>
	</blockquote>

	<ul>
		<li>Moe</li>
		<li>Larry</li>
		<li>Curly</li>
	</ul>
  • 不要な空白を入れない

	<!-- NG -->
	<p>実は不要な空白が、タグの後ろと次の行にあります・・・。</p> ←

title要素

ユーザビリティ、アクセシビリティ、SEOの観点から適切なものを設定します。
あまり長すぎるとGoole検索結果でカットされることになるので注意が必要です。内容によっては改変された文言が表示されることもあるようです。
内容はディレクターに確認してください。

  • ページ固有の内容を設定する必要があります。
  • ページ名、親カテゴリ名、サイト名の順に「｜（全角縦棒）」で区切ります。

	<title>ページ名｜サイト名</title>
	<title>ページ名｜カテゴリ名｜サイト名</title>

meta要素（keywords、description）

ユーザビリティ、アクセシビリティ、SEOの観点から適切なものを設定します。
ページ固有の内容を設定する必要があります。内容はディレクターに確認してください。


	<meta name="description" content="xxxxx">
	<meta name="keyword" content="xxx,xxxx,xxxxx">

meta要素（format-detection）

電話番号等の自動リンク機能を制御する為にformat-detectionを指定します。


	<meta name="format-detection" content="telephone=no">

meta要素（ogp）

SNS対応にOGP（Open Graph Protocol）を設定します。

  • head要素にprefixを追加します。

	<head prefix="og: http://ogp.me/ns#  article: http://ogp.me/ns/article#">

	記事ページはarticleでいいのですがトップページはwebsiteが正しいのでトップページのみ
	website: http://ogp.me/ns/website#としてください。
  • 特に理由がない限り、ogpの画像名をogp.pngとします。

	<meta property="og:title" content="xxxxx">（必須）
	<meta property="og:type" content="xxxxx">（必須）
	<meta property="og:url" content="xxxxx">（必須）
	<meta property="og:image" content="img/common/ogp.png">（必須）
	<meta property="og:description" content="xxxxx">
	<meta property="og:site_name" content="xxxxx">

meta要素（viewport）

  • Webサイトを適切なサイズで表示するよう必要に応じてviewportを設定します。

	<!-- デバイスの横幅に合わせる場合 -->
	<meta name="viewport" content="width=device-width,initial-scale=1.0">

	<!-- PCサイトをスマホ画面内に納めたい場合 -->
	<meta name="viewport" content="width=固定値, user-scalable=1">
  • 基本的にレスポンシブ対応で、ブレイクポイント（768px）を一つだけ設定した場合、タブレット版を作らずPC版を表示させる場合はviewport.jsを使用する

favicons

ブラウザ向け、スマホ向けの最低限のファビコンを設置します。
マルチアイコンや、各サイズに合わせて複数作成することも可能です。

  • 特に指定がない場合、以下の画像名で作成します。
  • faviconのNot Found対策として、root直下にfavicon.icoを置くようにします。

favicon.ico（32px×32px）
apple-touch-icon.png（144px×144px）


	<!-- favicon -->
	<link href="/favicon.ico" rel="shortcut icon" />

	<!-- apple-touch-icon-->

CSSファイルの読み込み

  • CSSは全て外部ファイル化し、<link>タグでCSSファイルを読み込みます。
  • CSSをHTMLに直接書き込む事は原則として禁止します。
  • CSSのtype属性は省略する。

	<!-- NG -->
	<link rel="stylesheet" href="//www.google.com/css/maia.css" type="text/css">

	<!-- OK -->
	<link rel="stylesheet" href="//www.google.com/css/maia.css">

webfontの読み込み

  • Font Awesomeを利用する場合は、fontawsome.jsを使用する
  • googleフォントを利用する場合は、重くならないように種類・ウエイト数を増やさない。

	<!-- googleフォント : Open Sansの400と800を読み込んでいる-->
	<link rel="stylesheet" href="//fonts.googleapis.com/css?family=Open+Sans:400,800" />

jsファイルの読み込み

  • JavaScriptはHEAD内、または</body>タグの直前に記述する
  • JavaScriptのtype属性は省略する。

	<!-- NG -->
	<script src="js/jquery-3.5.1.min.js" type="text/javascript"></script>

	<!-- OK -->
	<script src="js/jquery-3.5.1.min.js"></script>

jQueryの読み込み

  • 基本的ダウンロードして使用します。中のjsをなるべく最新verとして使用します。

	<!-- jQuery -->
	<script src="js/jquery.min.js"></script>

スクリプト

			$('.fontsize dd').removeClass('active');
			$(this).addClass('active');

			$.cookie('fontSize', thisSize, { path: '/'});
			$.cookie('opened',thisIndex, { path: '/'});

			changeSize(thisSize);
		});
	});

CSSハックについて

CSSハックを使う前に別の方法を試してみること。

  • CSSの定義は原則として外部CSSに記述してclass及びidで呼び出す
  • HTMLに直接記述する事は原則として禁止する

	/* NG */

formのスタイル

フォームの要素については、全てのデバイスでの検証が難しいため、できるだけ無難なスタイリングをする必要があります。

  • 最低限のリセットを設定する。

	/* --------------------------------------------------
	form
	-------------------------------------------------- */

	button, fieldset, form, input, label, legend, select, textarea {
	font-family: inherit;
	font-size: 100%;
	font-style: inherit;
	font-weight: inherit;
	margin: 0;
	padding: 0;
	vertical-align: middle;
	}

	fieldset {
	border: none;
	display: block;
	}

	label {
	cursor: pointer;
	}

	textarea {
	overflow: auto;
	}
  • 各要素のデフォルトスタイルを削除する。

	input, button, textarea, select {
		margin: 0;
		padding: 0;
		background: none;
		border: none;
		border-radius: 0;
		outline: none;
		-webkit-appearance: none;
		-moz-appearance: none;
		appearance: none;
	  }
  • IEのinput要素に出てくるマークを消去

	/* text */
	input::-ms-clear {
		visibility: hidden;
	}

	/* password */
	input::-ms-reveal {
		visibility: hidden;
	}

印刷対応

  • CSSファイルを別に作ってlink media="print"で読み込む。

	<link rel="stylesheet" type="text/css" media="print" href="print.css">
  • 印刷時に非表示にしたい要素やコンポーネントを指定。

	#header, #footer, #SideBar,等など{display:none;}
  • 印刷ボタンの設置

	<a onclick="window.print(); return false;" href="javascript:void(0)">このページを印刷する</a>
  • 改ページ

	.sheet  {
		page-break-after: always; //その位置で強制的に改ページさせる
		page-break-after: avoid; //その位置での改ページを避ける
	}
